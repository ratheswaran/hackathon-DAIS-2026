# Resonance Analytics — brand assets (for building `ra_template.pptx`)

These are the reusable brand visuals from the RA Design System Deck,
extracted so you can drop them straight into a PowerPoint template. The
`compose_deck` renderer draws all of this **natively** in from-scratch
mode — you only need these assets if you're hand-building a template (see
`../templates/README.md`).

> **A built template already ships** at `../templates/ra_template.pptx`,
> generated by `../templates/build_ra_template.py`, which embeds the PNGs
> below (cover poster, wordmarks, geo-mark) into the slide layouts. Edit that
> builder + re-run it rather than hand-assembling from these assets.

## Does PowerPoint support SVG? — Yes

Modern PowerPoint (2016+, and Microsoft 365 / PowerPoint for Mac) inserts
SVG natively: **Insert → Pictures → This Device → pick the `.svg`**. SVGs
stay vector-crisp at any size and are **recolorable** — right-click the
image → **Convert to Shape**, then restyle fills like any PowerPoint
shape. Web/older 2013 builds don't support SVG; use the PNG twins below.

**Rule of thumb used here:**
- **Vector marks** (brand-mark, geo-mark) ship as **SVG** (+ PNG twin) —
  crisp and recolorable.
- **Gradient/blur backgrounds** (the cover orb) ship as **high-res PNG** —
  CSS radial gradients + blur don't round-trip to SVG cleanly.

## Official RA logo (the "reso" radial mark) — used by the template

The **official** Resonance Analytics logo is a radial cluster of crimson
(`#F62459`) + light-blue (`#89C4F4`) dots around the "reso" mark. Source SVGs:
`resonance-logo.svg` (white "re" text, for **dark** bg) and
`resonance-logo-light.svg` (ink "re" text, for **light** bg). These are the
brand identity; the deck/chart palette stays **cobalt** (the logo is a brand
stamp, not a recolour — confirmed call).

| File | What it is | Use |
|---|---|---|
| `ra-logo-{dark,light}.png` | 643×640, the trimmed "reso" mark only (rsvg + PIL bbox trim) | mark-only placements |
| `ra-lockup-{dark,light}.png` | 2733×320, **mark + "Resonance Analytics"** (Manrope SemiBold), composed with PIL | the wordmark across the template (cover/section/closing top-left + content footers) |

`build_ra_template.py` places `ra-lockup-{dark,light}.png` wherever the old
cobalt-square `wordmark-*.png` sat. To regenerate after editing the SVGs, re-run
the rsvg-trim + PIL-compose steps (see the build guide).

## Bundled fonts (`fonts/`) — embedded into the template

`fonts/` holds the static TTFs (all SIL OFL → embeddable): Source Serif 4
(Regular/Bold/Italic/BoldItalic), Manrope (Regular/Bold), JetBrains Mono
(Regular/Bold/Italic). `templates/embed_fonts.py` embeds these into
`ra_template.pptx` so the serif + mono render on machines without them
installed. ~1.6 MB on disk; embedded full (not subset).

## Recoloring an SVG in PowerPoint

1. Insert the SVG (Insert → Pictures → This Device).
2. Right-click → **Convert to Shape** (it may become a group — Ungroup if
   you want individual pieces).
3. Select a piece → **Shape Fill** → set the brand hex (below).

The geo-mark's two amber elements (`stroke` on the ring, `fill` on the arc
+ dot) are both `#DF9B44` — change both to recolor it (e.g. to cobalt
`#254BB2` for a light-background variant).

## Fonts (install before editing the template)

- **Manrope** — body, UI, titles · https://fonts.google.com/specimen/Manrope
- **Source Serif 4** — display: covers, hero numbers, section titles,
  pull-quotes (load the *italic* axis too) · https://fonts.google.com/specimen/Source+Serif+4
- **JetBrains Mono** — page numbers / small data · https://fonts.google.com/specimen/JetBrains+Mono

In PowerPoint, **File → Options → Save → Embed fonts in the file** so the
template (and decks built from it) render faithfully on machines without
the fonts.

## Things that are just *type* (recreate directly, no asset needed)

- **Ghost section numeral** (the giant faint "02" on section dividers):
  a Source Serif 4 numeral, white at ~8–10% opacity, ~520pt, anchored
  top-right.
- **Eyebrow + dash**: Manrope 700, 13px, 0.16em tracking, UPPERCASE,
  cobalt on light / amber on dark, with a 28×2px dash to its left.
- **Corner mark** (top-right `RA · 06 · TYPE` label): JetBrains Mono,
  11px, 0.12em tracking, muted.

## Reproducing / tweaking these assets

The PNGs were rendered from small HTML/SVG sources (cover gradient + orb
from the design's exact CSS; marks from the SVGs in this folder). To
regenerate after an edit: render the SVGs with `rsvg-convert -w <px>` and
the HTML backgrounds with headless Chrome
(`--screenshot --window-size=1920,1080`, add
`--default-background-color=00000000` for transparency).
